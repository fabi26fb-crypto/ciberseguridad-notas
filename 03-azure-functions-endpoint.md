# Lab 03: Build a Basic Website Endpoint with Azure Functions

**Platform:** Microsoft Learn  
**Duration:** ~36 min | 8 units | 900 XP  
**URL:** https://learn.microsoft.com/en-us/training/modules/guided-project-build-basic-website-endpoint-with-functions/

---

## What I Did
- Created an Azure Function App in the Azure Portal
- Built an HTTP-triggered function that returns a response
- Tested the function endpoint via browser and HTTP requests
- Validated responses from the serverless function

## What I Learned
- What serverless computing means — no server management, pay per execution
- How Azure Functions are triggered (HTTP, Timer, Blob, Queue, etc.)
- How to create and deploy a basic function from the Portal
- The concept of Function App as the hosting container for functions

## Key Concepts
| Concept | Description |
|---------|-------------|
| Serverless | Run code without managing infrastructure |
| Azure Functions | Event-driven, serverless compute service |
| HTTP Trigger | Function activated by an HTTP request |
| Function App | Container that hosts one or more functions |
| Consumption Plan | Pay only when function executes |

## Tools Used
- Azure Portal
- Azure Functions
- HTTP trigger

## Relevance to AZ-900
- Azure compute services (serverless)
