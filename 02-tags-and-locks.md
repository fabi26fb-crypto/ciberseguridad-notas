# Lab 02: Organize and Protect Resources with Tags and Locks

**Platform:** Microsoft Learn  
**Duration:** ~36 min | 8 units | 900 XP  
**URL:** https://learn.microsoft.com/en-us/training/modules/guided-project-organize-resources-tags-locks/

---

## What I Did
- Applied tags to Azure resources for organization (environment, department, cost center)
- Created resource locks to prevent accidental deletion
- Tested the lock by attempting to delete a protected resource
- Managed resources through Azure Portal and resource groups

## What I Learned
- How tags work as key-value metadata for organizing and filtering resources
- Difference between **ReadOnly** and **Delete** locks
- How locks cascade from resource group to child resources
- Why tagging is critical for cost management and governance in real environments

## Key Concepts
| Concept | Description |
|---------|-------------|
| Tags | Key-value pairs for organizing resources (e.g. `env: production`) |
| Delete Lock | Prevents resource deletion but allows modifications |
| ReadOnly Lock | Prevents both deletion and modifications |
| Resource Group | Logical container for related Azure resources |

- Resource Groups
