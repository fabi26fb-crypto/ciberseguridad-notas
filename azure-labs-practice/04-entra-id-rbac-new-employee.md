# Lab 04: Configure New Employee Access (Entra ID & RBAC)

**Platform:** Microsoft Learn  
**Duration:** ~36 min | 8 units | 1100 XP  
**URL:** https://learn.microsoft.com/en-us/training/modules/guided-project-new-employee-access/

## What I Did
- Created a new user in Microsoft Entra ID (formerly Azure Active Directory)
- Assigned a role using Role-Based Access Control (RBAC)
- Applied the principle of least privilege to limit access
- Verified the user could only access what was explicitly granted

## What I Learned
- How Microsoft Entra ID manages identities in Azure
- How RBAC controls who can do what on which resources
- The principle of least privilege — give only the minimum access needed
- Difference between built-in roles: Owner, Contributor, Reader
- Why IAM is critical in cloud security environments

## Key Concepts
| Concept | Description |
|---------|-------------|
| Microsoft Entra ID | Cloud-based identity and access management service |
| RBAC | Role-Based Access Control — assigns permissions via roles |
| Least Privilege | Users get only the access they need, nothing more |
| Owner | Full access including managing permissions |
| Contributor | Can manage resources but not assign roles |
| Reader | View-only access |
